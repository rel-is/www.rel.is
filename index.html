rel is for all of us

  • rel is a free protocol
  • rel messages are ephemeral
  • rel is distributed
  • rel is easy to get started
  • rel is suited to everyday life
  • rel is ideal for push-notification-based scenarios, like alerts

rel is for the web

Messages are expressed as three URIs, so you can describe flexible, rich meaning and take advantage of existing web technologies.

Rel is a protocol

Rel describes how rel messages are passed from sender to recipient. Rel Exchanges (RXs) can pass messages around and receive incoming messages. Message Handlers can process rel messages and perform some sort of action in response.

rel is an early alpha

The specification is still being drafted.
